## Why

The Intelligent Service currently uses Redis Pub/Sub for fuzzy logic evaluation tracking, which introduces unnecessary complexity and potential desync with the edge nodes. The system must be updated to exclusively use the MQTT broker for telemetry ingestion and actuation commands. Furthermore, commands generated by the fuzzy logic must be persistently stored in the database for future auditing and analysis. Additionally, soil `ec` and `ph` sensors are out of scope for fuzzy evaluation and should be ignored during fuzzy inference. 

## What Changes

- **Remove Redis Pub/Sub**: The Intelligent Service will no longer use Redis for fuzzy logic eventing.
- **Direct MQTT Integration**: The fuzzy logic process will subscribe to the standard telemetry MQTT topic (`orchid/{device_id}/telemetry`) to receive real-time updates.
- **Fuzzy Evaluation Filter**: The fuzzy logic will process all environmental and soil parameters from the telemetry payload EXCEPT `ph` and `ec` (soil electrical conductivity).
- **Command Publishing**: Once fuzzy inference completes, the actuation result is published directly back to the MQTT broker on the command topic (`orchid/{device_id}/command/{actuator}`).
- **Command Logging via Ingestion**: The Ingestion Service will subscribe to the command topic (`orchid/+/command/+`) and persist the fired commands into a new database table `command_log` (or `command_history`).

## Capabilities

### New Capabilities
- `fuzzy-event-pipeline`: Defines the end-to-end MQTT event flow for fuzzy logic evaluation, command dispatch, and database persistence tracking.

### Modified Capabilities

## Non-goals

- Altering the mathematical logic of the fuzzy membership functions (only the transport/trigger mechanism is changing).
- Migrating existing TimescaleDB structures (only adding the new command log table).

## Impact

- `database/prisma/schema.prisma` will require a new `CommandLog` model.
- `services/intelligent-service` will remove Redis dependencies and add MQTT subscriber/publisher functionality.
- `services/ingestion-service` will add an MQTT subscription for command topics and DB insertion logic.
